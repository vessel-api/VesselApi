# Security Policy

## Reporting a Vulnerability

We take security seriously at VesselAPI. If you discover a security vulnerability, please report it responsibly.

**DO NOT** open a public GitHub issue for security vulnerabilities.

### How to Report

**Email:** support@vesselapi.com

**Please include:**
- Description of the vulnerability
- Steps to reproduce the issue
- Potential impact assessment
- Any suggested fixes (optional)
- Your contact information for follow-up

### What to Expect

| Timeline | Action |
|----------|--------|
| Within 48 hours | We'll acknowledge receipt of your report |
| Within 7 days | We'll provide an initial assessment |
| Varies by severity | We'll work on a fix and coordinate disclosure |

### Severity Levels

- **Critical**: Data breach, authentication bypass, remote code execution
- **High**: Privilege escalation, significant data exposure
- **Medium**: Limited data exposure, denial of service
- **Low**: Minor issues with limited impact

## Scope

This security policy covers:

| Service | URL |
|---------|-----|
| VesselAPI REST API | api.vesselapi.com |
| VesselAPI Dashboard | dashboard.vesselapi.com |
| VesselAPI Website | vesselapi.com |

## Out of Scope

The following are **not** covered by this policy:

- Third-party services and integrations
- Social engineering attacks against employees
- Physical security of offices or data centers
- Denial of service (DoS/DDoS) attacks
- Spam or phishing attempts
- Issues in third-party dependencies (report to upstream)

## Safe Harbor

We consider security research conducted in accordance with this policy to be:

- Authorized and legal
- Helpful to our security posture
- Exempt from legal action from us

To qualify for safe harbor:
- Report vulnerabilities promptly
- Avoid accessing or modifying user data
- Don't disrupt our services
- Don't publicly disclose before we've addressed the issue
- Act in good faith

## Recognition

We appreciate security researchers who help keep VesselAPI secure. With your permission, we'll:

- Acknowledge your contribution publicly
- Add you to our security hall of fame (coming soon)

## Questions

For non-security questions, please use:
- General support: support@vesselapi.com
- GitHub Issues: For bugs and feature requests

Thank you for helping keep VesselAPI and our users safe!
